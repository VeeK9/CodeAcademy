API

Application programming interface - jusu uzklausa is aplikacijos (front-end) issiunciama i serveri (back-end) ir jeigu viskas gerai - grazinami duomenys pagal jusu uzklausa, bet jei kazkas negerai - grazinama atitinkama error'o zinute.

JSON

JSON - Java Script Object Notation yra skirtas paversti duomenis i toki formata, kuris galetu buti siunciamas su API.
JSON formatas nuo JS object formato skiriasi tuo, kad raktiniai zodziai turi buti kabutese, o pacios kabutes visada dvigubos.

Tam, kad paversti JS sintakse i JSON formata arba atvirskiai, naudojamos dvi JSON komandos:
JSON.stringify(JS syntax) - is JS sintakses pavercia ir grazina JSON sintakses stringa.
JSON.parse(JSON syntax) - is JSON sintakses pavercia ir grazina JS sintakse (kaip objekta ar masyva).

Fetch

Funkcija, kurios pagalba mes galime siusti uzklausas i serveri ir gauti kazkokius atsakymus.

Festch vykdymo eiga:

  1. Fetch'inant (siunciant uzklausa) nurodomas URL, galima nurodyti ir papildomas opcijas. Tarp tu opciju yra metodai:
    • GET - gauti info
    • POST - pateikti info
    • DELETE - trinti info
    • PATCH - redaguoti info
    • PUT - redaguoti info
  2. Apdoroti gauta info (vykdosi kai nera error)
  3. Naudoti gauta info (vykdosi kai nera error)
  4. Pagaunam (catch) ir kazka atliekam su tuo error (vykdosi tik error atveju)
  5. Galiausiai (finally) dar kazka atliekam (vykdosi bet kokiu atveju)
