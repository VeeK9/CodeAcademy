Session Storage vs Local Storage vs Cookies

Duomenu saugijimas vartotojo dalyje

Cookies, Session storage ir Local storage informacija yra saugojama jusu irenginyje/aplikacijoje/narsykleje...
Skirtingose svetainese/narsyklese/irenginiuose cookies/session/local storage yra saugojama atskirai.

Visi lokaliai saugojami duomenys turetu buti string pavidalu!!!

Cookies

Saugojama iki 4kb.
Reikia rankiniu budu nustatyti iki kada duomenys yra saugojami.
Siunciami kartu su request'ais i back-end'a.

Session

Laiko duomenis aplikacijoje/narsykleje, per kuria atidaroma kazkuri svetaine, kuriai tie duomenys priskiriami.
Kai pasibiagia sesija (kai isjungiama svetaine) sitie duomenys yra istrinami.

Daznai naudojamas saugoti JWT token

Storage

  • sessionStorage.length - grazina issaugotu elementu kieki
  • sessionStorage.setItem('raktas', 'reiksme')
  • sessionStorage.getItem('raktas') - grazina reiksme pagal rakta
  • sessionStorage.removeItem('raktas')
  • sessionStorage.clear() - istrina viska is sessionStorage
  • sessionStorage.key(index) - grazina raktini zodi esanti atitinkamoje indekso pozicijoje.

Local

Laiko duomenis aplikacijoje/narsykleje, per kuria atidaroma kazkuri svetaine, kuriai tie duomenys priskiriami.
Duomenys yra issaugojami per daugeli sesiju. Duomenys gali buti istrinami tik naudojant atitinkamus metodus (kode) arba per dev tools.

Storage

  • localStorage.length - grazina issaugotu elementu kieki
  • localStorage.setItem('raktas', 'reiksme')
  • localStorage.getItem('raktas') - grazina reiksme pagal rakta
  • localStorage.removeItem('raktas')
  • localStorage.clear() - istrina viska is localStorage
  • localStorage.key(index) - grazina raktini zodi esanti atitinkamoje indekso pozicijoje.
