HTML formu validavimas

Formu validavimas - tikrinimas ar formos ivesciu laukai teisingi supildyti.
Formu validavimas yra reikalingas tam, kad nebutu leidziama pateikti formos, jeigu kazkokie laukai neteisingai supildyti arba isvis nesupildyti.

  • required - ivesties laukas privalomas
  • minlength - minimalus simboliu kiekis - text/password/url/email
  • maxlength - maksimalus simboliu kiekis - text/password/url/email
  • min - maziausias galimas skaicius - number.range.date
  • max - didziausias galimas skaicius - number.range.date
  • step - skaiciaus didejimo/mazejimo zingsnis - number/range/date
  • pattern - specifine seka (su regex) - password/tel
  • accept - kokio tipo faila/us priima - file
  • multiple - leidimas kelti ivesciu i viena - file/email

DOM - Document Object Model

Node

Node'as - HTML elementas JS'o aplinkoje

Galima HTML pildyti innerHTML'u - rasant HTML koda JS'o string'e ir si iklijuoti HTML dokumenta, kur to stringo viduje esantis HTML kodas bus skaitomas kaip HTML ir atvaizduojamas vartotojui.
Bet sitas budas turi daug minusu, pradedant nuo saugumo, baigiant implementacija.

Geresnis budas pildyti HTML'a su JS - kurti HTML elementus (node'us) JS'e ir juos append'inti i kazkuri HTML'o elementa.
Node'ai yra kuriami ir modifikuojami naudojant Node'u metodus.

Node'u metodai

  • document.createElement('tagVardas')
  • document.createTextNode('kazkoks tekstas...')
  • kam.append(ka0, ka1, ka2, ...) - i koki node'a iterpi koki/kokius node'a/node'us/tekstus.
  • kam.appendChild(ka) - i koki node'a iterpi VIENA kita NODE'a
  • kam.prepend(ka0, ka1, ka2, ...) - tas pats kas append, tik prideda i pradzia, o ne gala
  • kam.setAttribute('name', 'value')
  • kam.classList
    • add('vienaKlase', 'kitaKlase', ...)
    • remove('viena klase', 'kitaKlase', ...)
    • replace('klaseIsimi', 'klasePridedi')
    • toggle('klase')
